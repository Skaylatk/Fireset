# Contributing to This Engine

Thank you for your interest in contributing! Please follow these guidelines:

1. **Make Changes**  
   - You can work directly on the `main` branch of your fork.  
   - Focus on **one feature, bugfix, or improvement per PR**.  
   - Keep your code clean, readable, and consistent with the existing style.  
   - Avoid breaking existing functionality.

2. **Documentation**  
   - Update Doxygen comments for any new or modified functions.  
   - Update usage examples or README if your changes affect them.

3. **Testing**  
   - Test your changes locally.  
   - Ensure existing examples or test programs still compile and run.  
   - Include new test code for added features when possible.

4. **Pull Request**  
   - Push your changes to your fork.  
   - Open a PR against the `main` branch of the official repository.  
   - Include a clear description of your changes and link any related issues.

> Following these steps helps maintain code quality and makes it easier to review and merge contributions. Thank you for improving this engine!
